=======================================================

Current issue:

Swipe-to-back is inconsistent.

Some sections support it.

Some sections do not.

Some nested screens incorrectly exit directly to Hub.

Some screens restore bottom nav late or incorrectly.

This must be fixed globally.

Required behavior:

Every screen that has a previous internal screen/state must support Android swipe-to-back correctly.

This applies to:

* Hub nested screens
* Chordex
* Chordex Discover
* Chordex Practice
* Chordex song/detail screens
* Stagex
* Stagex nested/editor states
* Drumex
* Drumex nested panels
* Groovex
* Groovex nested panels
* Vocalex
* Vocalex nested panels
* settings
* dialogs
* bottom sheets
* modals

Swipe-to-back must follow hierarchy:

1. If a modal/dialog/sheet is open:

   * close that first.

2. Else if a nested screen is open:

   * go back to the previous screen inside the same app.

3. Else if user is on the app root screen:

   * apply the global swipe-back setting:

     * either return to Hub
     * or do nothing / require manual back

4. Never skip from a nested screen directly to Hub unless the user explicitly uses an exit-to-Hub control.
