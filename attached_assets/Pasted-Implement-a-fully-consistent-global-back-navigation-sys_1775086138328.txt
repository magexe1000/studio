Implement a fully consistent global back navigation system across the entire app, including proper Android predictive back support and logical navigation behavior.

IMPORTANT:
- Do NOT break existing navigation
- Must work across ALL sections and nested views
- Must feel native, smooth, and predictable
- Must be optimized and bug-free

----------------------------------
1. GLOBAL BACK BEHAVIOR (CORE)
----------------------------------

Define a clear navigation hierarchy:

- Main sections (root level):
  - Songs
  - Chords
  - Settings
  - Library

- Nested views:
  - Song details
  - Chord editor
  - Custom chord builder
  - Modals / panels

BEHAVIOR:

- If user is inside ANY nested view:
  → Back action returns to previous view

- If user is inside a modal/panel:
  → Back closes the modal FIRST

- If user is in a sub-section (e.g. inside a song):
  → Back returns to main section

----------------------------------
2. ROOT LEVEL EXIT LOGIC
----------------------------------

When user is at root (main sections):

- First back press:
  → Show message:
    "Press back again to exit"

- Second back press (within ~2 seconds):
  → Exit app

----------------------------------
3. PREDICTIVE BACK (ANDROID)
----------------------------------

Implement Android predictive back support:

- Use modern navigation handling
- Allow smooth preview gesture
- Ensure transitions match back navigation

REQUIREMENTS:
- Smooth animation
- No broken states
- Must reflect actual navigation stack

----------------------------------
4. NAVIGATION STACK SYSTEM
----------------------------------

Create a centralized navigation stack:

- Track history of views
- Push on navigation
- Pop on back

DO NOT rely only on UI state

----------------------------------
5. MODALS & PANELS PRIORITY
----------------------------------

Back priority order:

1. Close open modal/panel
2. Exit nested view
3. Navigate to previous section
4. Exit app (double back)

----------------------------------
6. CHORDS / EDITOR FIX (CRITICAL)
----------------------------------

Ensure back works inside:

- Chord builder
- Song editor
- Live mode
- Any detailed view

No section should trap the user

----------------------------------
7. WEB SUPPORT (BROWSER)
----------------------------------

Also support:

- Browser back button
- Sync with history API

Use:
- pushState
- popstate listener

----------------------------------
8. PERFORMANCE
----------------------------------

- No lag when pressing back
- Instant response
- No unnecessary re-renders

----------------------------------
9. UX DETAILS
----------------------------------

- Smooth transitions
- No flicker
- Consistent behavior everywhere

----------------------------------
GOAL
----------------------------------

Create a fully reliable and intuitive back navigation system that works across the entire app, supports Android predictive back gestures, and ensures users can always navigate backwards correctly without confusion or dead-ends.
