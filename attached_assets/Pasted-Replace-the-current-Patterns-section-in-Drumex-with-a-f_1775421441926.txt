Replace the current "Patterns" section in Drumex with a fully functional and useful "Library" section.

IMPORTANT:
- Remove the existing Patterns section completely
- Replace it with a new "Library" section
- Keep UI clean, minimal, and consistent with the rest of the app
- Must be fast, responsive, and actually useful (not empty)

----------------------------------
1. PURPOSE OF LIBRARY
----------------------------------

The Library must provide ready-to-use drum content that users can quickly insert into their patterns.

----------------------------------
2. STRUCTURE
----------------------------------

Organize the Library into categories:

- Grooves
- Fills
- Basic Beats

Optional:
- Genre (Rock, Pop, Funk, etc.)

----------------------------------
3. CONTENT DESIGN
----------------------------------

Each item in the library must:

- Have a name (e.g. "Basic Rock Groove")
- Show a small visual preview (mini grid or simplified pattern)
- Be clean and readable

----------------------------------
4. INTERACTION
----------------------------------

Users must be able to:

- Tap → preview the groove (play audio)
- Tap "Insert" → apply to current pattern
- Optionally:
  - Insert at cursor position
  - Replace current measure

----------------------------------
5. AUDIO PREVIEW
----------------------------------

- Each groove must have instant playback
- No lag when previewing
- Use optimized audio system (reuse buffers)

----------------------------------
6. INSERT LOGIC
----------------------------------

When inserting:

- Align with current BPM
- Respect grid timing
- Do NOT break existing pattern unless user chooses replace

----------------------------------
7. UI DESIGN
----------------------------------

- Use card-based layout
- Minimal and clean
- Smooth scrolling
- No clutter

Each item example:

[ Groove Name ]
[ mini preview ]
[ Play ] [ Insert ]

----------------------------------
8. PERFORMANCE
----------------------------------

- Lazy load library content
- No heavy load at startup
- Instant interaction

----------------------------------
9. FUTURE READY
----------------------------------

Structure system to allow:

- Adding more grooves easily

10. REMOVE OLD SYSTEM

- Completely remove unused "Patterns" logic

11. FINAL RESULT
----------------------------------

The Library must:

- Be useful immediately

GOAL

Create a clean, fast, and practical drum Library that replaces the useless Patterns section and provides real value to the user.
