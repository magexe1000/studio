Implement OTA update version checking using the app version displayed in Studio Settings (bottom section) as the single source of truth.

GOAL:
Ensure that the OTA system compares the remote version against the exact version currently shown in Studio → Settings → bottom (App Version), instead of using any hardcoded or duplicated version values.

---

1. SOURCE OF TRUTH (CRITICAL)

---

* Use the version value currently displayed in:
  Studio → Settings → bottom section (App Version)

* This value must be:

  * centrally stored (e.g. global config / constants / app state)
  * accessible from both UI and OTA logic

* Remove any duplicated or hardcoded version strings across the app

---

2. OTA VERSION CHECK LOGIC

---

* Fetch remote version from:
  (e.g. /version.json or remote config)

Example:
{
"version": "1.2.3",
"changelog": "...",
"mandatory": false
}

* Compare:
  remoteVersion vs currentAppVersion (from Settings)

* If remoteVersion > currentAppVersion:

  * trigger update available state

---

3. VERSION SYNC REQUIREMENT

---

* The version shown in Settings MUST always match:

  * the internal app version
  * the OTA comparison version

* If version changes:

  * it should automatically reflect in Settings UI
  * without manual duplication

---

4. UI BEHAVIOR

---

* When update is available:

  * show minimal indicator (top-right in Studio)

* After updating:

  * on next app launch:

    * display changelog modal

---

5. STRUCTURE REQUIREMENT

---

* Create a centralized version provider/module:
  e.g.:

  * appConfig.version
  * useAppVersion()

* OTA system must ONLY read from this source

---

6. CLEANUP

---

* Remove:

  * old version checks
  * duplicate constants
  * unused version variables

---

7. EDGE CASES

---

* If version cannot be fetched:

  * fail silently (no crash)

* If version format differs:

  * normalize before comparing (semver)

---

## GOAL

Make OTA updates reliable by ensuring the app always compares against the same version that the user sees in Studio Settings.
