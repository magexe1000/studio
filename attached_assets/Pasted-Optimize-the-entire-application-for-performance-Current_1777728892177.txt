Optimize the entire application for performance. Current load time (30–60 seconds) is unacceptable. The app must load fast, feel instant, and be smooth across all sections.

IMPORTANT:
Do NOT apply superficial optimizations. Perform deep analysis and refactor where necessary.

---

1. ROOT CAUSE ANALYSIS

---

Identify exactly why the app is slow:

* Large bundle size
* Blocking scripts during startup
* Unoptimized assets (images, audio, fonts)
* Excessive API calls on load
* Inefficient state management
* Unnecessary re-renders
* Heavy libraries
* Sync processes blocking UI
* Poor lazy loading strategy

Log and measure:

* Time to first render
* Time to interactive
* Total load time

---

2. INITIAL LOAD OPTIMIZATION

---

* Reduce initial bundle size aggressively
* Load ONLY critical components at startup
* Defer non-essential modules (Drumex, Stagex, etc.)
* Use lazy loading / dynamic imports

Goal:
Initial UI should render in under 1–2 seconds

---

3. CODE SPLITTING

---

* Split app by sections:

  * Studio (core)
  * Drumex
  * Chordex
  * Vocalex
  * Stagex

* Load each module ONLY when accessed

* Cache modules after first load

---

4. REMOVE DEAD CODE

---

* Remove:

  * unused components
  * old logic
  * duplicated functions
  * unused dependencies

* Audit all imports

---

5. ASSET OPTIMIZATION

---

* Compress images
* Lazy load media (audio, icons)
* Avoid loading large assets at startup
* Replace heavy formats if needed

---

6. STATE & RENDER OPTIMIZATION

---

* Prevent unnecessary re-renders
* Memoize components where needed
* Optimize global state usage
* Avoid deep nested updates

---

7. NETWORK OPTIMIZATION

---

* Minimize API calls on startup
* Batch requests if needed
* Cache responses
* Add timeout and fallback

---

8. BACKGROUND TASKS

---

* Move non-critical processes to background:

  * sync
  * version check (OTA)
  * preload assets

* Do NOT block UI

---

9. CACHING SYSTEM

---

* Cache:

  * user data
  * projects
  * settings

* Use local storage or indexedDB

* Avoid re-fetching unchanged data

---

10. LOADING EXPERIENCE

---

* Replace long loading screen with:

  * instant UI render
  * progressive loading

* Show skeletons instead of spinners

---

11. PERFORMANCE TARGETS

---

* First render: < 2 seconds
* Fully interactive: < 3 seconds
* No freezing or lag

---

12. CLEAN ARCHITECTURE

---

* Modularize code
* Separate concerns properly
* Ensure scalability

---

## GOAL

Transform the app into a fast, responsive, and smooth experience:

* Instant startup
* No long loading screens
* Efficient resource usage
* Professional-grade performance
